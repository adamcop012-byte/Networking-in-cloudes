1. SECURITY IMPROVEMENTS

CRITICAL FIX: CORS Vulnerability
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Problem:
  All three services had: app.add_middleware(CORSMiddleware, 
                                            allow_origins=["*"], ...)
  This allows ANY website to make requests to your API — security risk!

Solution:
  Changed to: allowed_origins = os.getenv("CORS_ORIGINS", "...")
  CORS origins now controlled via environment variables
  
  In docker-compose.yml:
    environment:
      - CORS_ORIGINS=http://localhost,http://localhost:80

Impact:
  ✅ Only allowed origins can access the API
  ✅ Configurable per environment (local/staging/production)
  ✅ Production deployment will specify domain only

Files Modified:
  • services/erp/main.py
  • services/crm/main.py
